# 📊 Swiggy Data Analysis using SQL

## 🔍 Project Overview
This project analyzes Swiggy restaurant data to extract meaningful business insights related to pricing, delivery performance, and customer satisfaction. The goal is to simulate real-world data analyst tasks using SQL.

---

## 🛠 Tools & Technologies
- MySQL
- SQL
- GitHub

---

## 📂 Project Structure
- `queries.sql` → Contains SQL queries used for analysis
- `insights.md` → Business insights derived from the data

---

## 📊 Dataset
Dataset sourced from Kaggle (Swiggy Restaurant Data).  
It includes information such as:
- Restaurant name
- Location (Area, City)
- Price
- Ratings
- Delivery time
- Food type

---

## 📈 Key Analysis Performed

### 🔹 Data Cleaning
- Renamed columns for better readability
- Standardized naming conventions (snake_case)
- Converted price column to appropriate numeric format

### 🔹 Exploratory Analysis
- Total number of restaurants
- Average delivery time
- City-wise restaurant distribution

### 🔹 Business Logic & Segmentation
- Categorized restaurants into Budget, Mid-range, and Premium
- Classified delivery speed (Fast, Moderate, Slow)
- Grouped restaurants based on rating performance

### 🔹 Insight Generation
- Analyzed relationship between price and ratings
- Studied impact of delivery time on customer satisfaction

---

## 📌 Key Insights
- Premium restaurants tend to have higher ratings, indicating better perceived quality
- Faster delivery times are associated with better customer ratings
- Delivery efficiency plays a critical role in customer satisfaction
- Mid-range restaurants dominate the dataset
